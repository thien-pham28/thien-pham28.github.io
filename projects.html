PURGATORIO

September 2022 - April 2023 | Unity2D

Description

An isometric roguelike pixel-art shooter inspired by the Divine Comedy. Play as a corrupt priest trapped in purgatory. The priest must fight through endless hordes of demons and confront the seven deadly sins to redeem his soul.

Features

  • Procedurally generated dungeon layouts
  • Random loot system with 8 weapon types, 9 weapon modifiers, creating more than 1500 possible weapon variants
  • Seven boss battles
  • Optional challenge events for extra loot

Project Type

University's Capstone project developed in a team over two semesters.

My Contributions

  • Designed and implemented the random weapon/loot system
  • Coded the weapons' shooting, reloading, swapping mechanics
  • Programmed the inventory and pickup/dropoff system
  • Developed the tutorial scene and created a simple dialogue system used both in the tutorial and boss fights
  • Created artwork and animations for some enemies and one boss (Envy) featured in the game.
  • Designed and implemented the challenge event system
  • Implemented currency
  • Designed and coded the Lust boss fight
  • Bug fixing and polishing other gameplay features

Project Insights

My main contribution in this project was the weapon system. I started out by making a list of weapon stats such as damage, ammo capacity, fire rate, and reload speed. I then used that to create a table featuring all weapon types and their values for each weapon stat. By estimating damage per second for each weapon type, I then tweak the values to ensure that each weapon can fill a different role while all remaining viable. Once the standard stats for each weapon are finalized, I created another table for modifiers that would alter those stats as well as special modifiers that can make bullets bouncy or change the firing mechanic. I then implemented the weapons using inheritence, with a parent 'weapon' class and mutliple child classes for different weapon features. This system allowed me to easily generate random weapons as loot. For the weapons themselves I applied mutliple techniques such as raycasting, vector manipulation, and circle casting to create different weapon features that can greatly alter gameplay.

My other major contribution is the game events system which utilizes event handling to create the dialogue system, the in-game events, and progress checkpoints.

AI Pathfinder

April 2022 | Unity3D

Description

An AI simulation of reinforcement learning in pathfinding. Train 2 AI agents to pick up and delivery objects to different locations on a customizable grid.

Features

  • Drawable map and customizable pickup/dropoff locations, capacity
  • Choose between 2 learning algorithm (SARSA, Q-learning) and 3 policies (PRANDOM, PGREEDY, PEXPLOIT)
  • Customizable values for movement speed, learning rate and discount rate
  • Heatmap that shows most used paths

Project Type

Solo project as part of a university course.

Project Insights

Implementing the learning algorithms were surprisingly the easiest part. The most difficult task was building and visualizing the world. First I had to build a custom grid map that would allow the user to draw cells and place agents and zones. The grid itself has access to the locations of both agents and the pickup/dropoff zones, however this information is not included in the algorithms. The algorithms only have access to the agent's location, their distance, their state and the state of the zones. Then I used the event handling system to build a tick-based timer to move the agents step-by-step. By applying the algorithms the agents could learn the most optimal paths as they move. Finally, I created UI to customize values, change algorithms, and policies. I also added auto-pausing mechanics and a heatmap for ease of use.
